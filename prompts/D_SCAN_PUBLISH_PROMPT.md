# D_scan_publish 发布模块 Prompt

> 本文档用于指导 AI 助手理解和维护发布模块。


---

## 一、模块定位

`D_scan_publish` 是项目的**发布模块**，负责将扫描结果渲染为图片并发送。

**核心职责**：
1. 读取 daily_scan 已导出的产物
2. 按 profile 配置渲染图片
3. 发送到指定平台

---

## 二、职责边界

### 2.1 核心职责

| 职责 | 说明 |
|------|------|
| 文件读取 | 读取 C_daily_scan 导出的 CSV/JSON 文件 |
| 图片渲染 | 按配置渲染榜单图片 |
| 消息发送 | 发送图片到指定平台 |

### 2.2 禁止事项

- ❌ **不参与评分**：评分由 B_scoring 负责
- ❌ **不读取数据库**：数据由 C_daily_scan 导出
- ❌ **不修改扫描结果**：只读取和展示

```

```

---

## 四、核心数据结构

### 4.1 PublishProfile（发布配置）

定义一次发布任务的配置：

| 字段 | 说明 |
|------|------|
| `name` | 配置名称 |
| `title` | 图片标题 |
| `columns` | 显示列配置 |
| `sort_by` | 排序字段 |
| `top_n` | 显示数量 |

### 4.2 PublishRequest（发布请求）

表示一次发布任务的输入：

| 字段 | 说明 |
|------|------|
| `profile_name` | 配置模板名称 |
| `csv_path` | 输入 CSV 文件路径 |
| `output_dir` | 图片输出目录 |
| `send_enabled` | 是否发送 |

### 4.3 PublishResult（发布结果）

表示一次发布任务的输出：

| 字段 | 说明 |
|------|------|
| `image_path` | 生成的图片路径 |
| `sent` | 是否已发送 |
| `message` | 发送结果消息 |

## 五、渲染设计

### 5.1 渲染流程

```
PublishRequest -> 加载 CSV -> 按 profile 过滤 -> 渲染图片 -> 保存 -> 发送（可选）
```

### 5.2 渲染配置

通过 `profiles.py` 定义不同的发布模板：
- 支持不同的列配置
- 支持不同的标题和样式
- 支持不同的排序规则

### 5.3 图片格式

- 输出 PNG 格式图片
- 支持自定义尺寸和样式
- 支持中文显示

---

## 六、发送设计

### 6.1 发送抽象

**设计原则**：
- 发送客户端通过配置初始化
- 不在代码中硬编码连接信息
- 支持多种发送渠道扩展

### 6.2 配置管理

**敏感信息保护**：
- 所有连接配置通过环境变量或配置文件管理
- 不在代码中暴露 Token、密码、URL
- 配置文件不纳入版本控制

---

## 七、运行方式

### 7.1 CLI 入口

```bash
python3 -m D_scan_publish --profile classic_a_share --csv-path output/daily_scan/scan_20260330.csv
```

### 7.2 参数说明

| 参数 | 说明 |
|------|------|
| `--profile` | 发布配置名称 |
| `--csv-path` | 输入 CSV 文件路径 |
| `--output-dir` | 图片输出目录 |
| `--send` | 是否发送 |
| `--no-send` | 不发送（仅生成图片） |

---

## 八、与扫描模块的关系

- D_scan_publish 依赖 C_daily_scan 的导出产物
- 两者通过文件解耦，不直接调用
- C_daily_scan 完成后，D_scan_publish 可独立运行

---

## 九、AI 助手行为准则

当基于此 Prompt 维护发布模块时，AI 助手应：

1. **保持展示边界**：不添加任何计算或筛选逻辑
2. **保护敏感信息**：不在代码中暴露发送配置
3. **通过配置文件管理连接**：不硬编码 URL、Token、密码
4. **维护渲染格式兼容**：不随意修改图片样式
5. **支持多渠道扩展**：发送客户端设计保持抽象
