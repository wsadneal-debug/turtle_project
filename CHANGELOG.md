### 📋 待办事项

#### 高优先级 🔴

- [ ] **股票名称解析**：当前报告中使用"某上市公司"占位符，需要关联真实股票代码和名称
- [ ] **定时任务配置**：设置 cron 定时任务实现自动运行
- [ ] **错误处理优化**：增强网络异常、页面结构变更的容错能力

#### 中优先级 🟡

- [ ] **多数据源整合**：合并东方财富和阿思达克的数据，去重处理
- [ ] **股票池配置**：支持自定义监控股票列表（如只监控恒生指数成分股）
- [ ] **推送通知**：发现高风险股票时自动推送提醒（邮件/消息）

#### 低优先级 🟢

- [ ] **历史数据对比**：记录股票风险历史，生成风险趋势图
- [ ] **Web 界面**：简单的 Web 仪表盘展示风险股票清单
- [ ] **API 接口**：提供 RESTful API 供其他系统调用
- [ ] **Docker 化**：容器化部署，简化安装流程

### 🐛 已知问题

1. **股票名称匹配**：新闻标题中的股票名称需要与港股代码准确关联
2. **假阳性识别**：部分市场风险关键词可能误判（如"封锁"可能指地理封锁而非股票风险）
3. **页面结构依赖**：爬虫依赖网站 HTML 结构，页面改版可能导致爬取失败

### 🔧 技术栈

- **语言**：Python 3.8+
- **浏览器自动化**：Chrome CDP (Chrome DevTools Protocol)
- **HTTP 请求**：requests, websocket-client
- **数据处理**：json, re (正则表达式)
- **报告生成**：Markdown 模板
