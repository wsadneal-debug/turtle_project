# B_scoring 评分模块 Prompt

> 本文档用于指导 AI 助手理解和维护评分模块。


---

## 一、模块定位

`B_scoring` 是项目的**纯公式模块**，负责股票评分计算。

**核心特点**：
- 不读数据库
- 不写数据库
- 只接受快照输入，返回评分结果
- 规则通过外部文件配置，支持热更新

---

## 二、职责边界

### 2.1 核心职责

| 职责 | 说明 |
|------|------|
| 评分计算 | 基于 SecuritySnapshot 计算各维度分数 |
| 规则解析 | 解析外部规则文件，构建评分配置 |
| 分档查分 | 根据分档表将指标值映射为分数 |

### 2.2 禁止事项

- ❌ **不依赖数据库连接**：禁止 import sqlite3 / sqlalchemy
- ❌ **不导入 repository / adapter / sync 模块**
- ❌ **不做数据加载**：数据加载由 B_compute 负责

## 四、核心数据结构

### 4.1 SecuritySnapshot（股票快照）

评分引擎的输入数据，包含股票的全部评估指标：


**单位约定**：
- 所有比率类字段：百分数形式（如 60.0 表示 60%）
- 金额类字段：元
- 股数类字段：股

### 4.2 ScoreResult（评分结果）

评分引擎的输出数据：

| 字段 | 说明 |
|------|------|
| `passed_tier1` | 是否通过第一层筛选 |
| `passed_tier15` | 是否通过预筛选 |
| `passed_tier2_hard` | 是否通过硬门槛 |
| `rankable` | 三层全部通过才为 True |
| `safety_score` | 安全维度分数 |
| `dividend_score` | 分红维度分数 |
| `valuation_score` | 估值维度分数 |
| `total_score` | 总分 |
| `fail_reasons` | 失败原因列表 |
| `labels` | 扩展标签（如弱周期、高质量） |

**关键约束**：
- `rankable=False` 时，各维度分数必须为 None，不得填 0
- `exclusion_reason` 记录主动排除原因（如行业排除），与 `fail_reasons` 分开

## 五、评分流程设计

### 5.1 Classic 评分流程

**设计概念**：

```
SecuritySnapshot -> Tier1 筛选 -> Tier1.5 预筛选 -> Tier2 硬门槛 -> 维度评分 -> 总分计算
```

**三层筛选设计概念**：
- **Tier 1**：基础资格筛选
- **Tier 1.5**：核心财务指标筛选
- **Tier 2**：深度财务指标硬门槛

**三维度评分设计概念**：
- **安全分**：评估财务安全性和稳定性
- **分红分**：评估分红能力和可持续性
- **估值分**：评估估值吸引力


## 六、规则配置设计

### 6.1 规则文件结构

规则通过外部文本文件配置，支持：
- 分档表定义（指标值区间 -> 分数）
- 筛选阈值定义
- 权重配置
- 版本标识

### 6.2 分档表查分设计概念

- 根据指标值查找对应分数区间
- 支持线性插值或离散映射
- 处理边界值和特殊值（如负值）

### 6.3 规则热更新

- 规则文件修改后无需重启服务
- 通过 `rules_version` 字段记录版本
- 支持多套规则并存（通过 `branch_key` 区分）

## 七、对外接口

### 7.1 Classic 评分接口

| 函数 | 说明 |
|------|------|
| `load_score_rules(path)` | 加载规则配置 |
| `score_stock(snapshot, rules)` | 单只股票评分 |
| `score_batch(snapshots, rules)` | 批量评分 |
| `lookup_band(value, bands)` | 分档表查分 |

## 八、工程约束

### 8.1 Import 边界检查

B_scoring 目录必须禁止以下 import：
- `sqlite3`
- `sqlalchemy`
- `repository` / `adapter` / `sync` 模块

### 8.2 CI 规则建议

建议在 CI 中加入扫描规则：
- 检查 B_scoring 目录的 import 语句
- 发现违规 import 时阻止合入

---

## 九、AI 助手行为准则

当基于此 Prompt 维护评分模块时，AI 助手应：

1. **保持纯公式边界**：不引入任何数据库依赖
2. **遵循单位约定**：比率类字段使用百分数形式
3. **维护规则可扩展性**：新规则通过外部文件配置
4. **保护敏感信息**：不在规则文件中暴露任何 Token 或密码
