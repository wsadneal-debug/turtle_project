# E_daily_pipeline 调度模块 Prompt

> 本文档用于指导 AI 助手理解和维护调度模块。


---

## 一、模块定位

`E_daily_pipeline` 是项目的**调度入口模块**，负责编排每日自动化流程。

**核心职责**：
1. 定时启动早间财报同步任务
2. 定时启动晚间股价同步、计算、扫描、发布流程
3. 每个节点完成/失败时发送通知

---

## 二、职责边界

### 2.1 核心职责

| 职责 | 说明 |
|------|------|
| 流程编排 | 定义和执行步骤序列 |
| 步骤调度 | 按顺序执行各模块 |
| 状态通知 | 发送流程进度通知 |

### 2.2 禁止事项

- ❌ **不包含业务逻辑**：具体计算由其他模块完成
- ❌ **不直接操作数据库**：数据操作由各模块负责
- ❌ **不修改评分规则**：规则由 B_scoring 管理

## 四、流程配置设计

### 4.1 PipelineConfig（流程配置）

定义一次流程执行的配置：

| 字段 | 说明 |
|------|------|
| `name` | 流程名称 |
| `steps` | 步骤列表 |
| `timeout` | 总超时时间 |

### 4.2 PipelineStep（步骤配置）

定义单个步骤的配置：

| 字段 | 说明 |
|------|------|
| `name` | 步骤名称 |
| `script` | 执行脚本/模块 |
| `args` | 脚本参数 |
| `required` | 是否必须成功 |
| `timeout` | 步骤超时时间 |
| `notify_on_start` | 开始时是否通知 |
| `notify_on_success` | 成功时是否通知 |
| `notify_on_failure` | 失败时是否通知 |

## 五、晚间流程设计

### 5.1 流程顺序

```
1. 港股通名单同步
2. A股行情同步
3. 港股行情同步
4. A股 Classic 计算
5. 港股通 Classic 计算
6. A股 Classic 扫描
7. 港股通 Classic 扫描
8. 发布确认
```

### 5.2 步骤依赖

- 同步步骤必须先完成
- 计算步骤依赖同步完成
- 扫描步骤依赖计算完成
- 发布步骤依赖扫描完成

### 5.3 失败处理

- `required=True` 的步骤失败会中断流程
- `required=False` 的步骤失败会继续执行
- 失败时发送通知

## 七、运行方式

### 7.1 晚间流程入口

```bash
cd E_daily_pipeline
python3 run_daily.py
```

### 7.2 CLI 入口

```bash
python3 -m E_daily_pipeline --dry-run
```

### 7.3 参数说明

| 参数 | 说明 |
|------|------|
| `--dry-run` | 试运行模式（不实际执行） |
| `--date` | 指定执行日期 |
| `--skip-sync` | 跳过同步步骤 |
| `--skip-publish` | 跳过发布步骤 |

---

## 八、通知设计

### 8.1 通知时机

| 时机 | 说明 |
|------|------|
| 流程开始 | 通知流程已启动 |
| 步骤开始 | 通知步骤已开始（可选） |
| 步骤成功 | 通知步骤已完成（可选） |
| 步骤失败 | 通知步骤失败（必须） |
| 流程完成 | 通知流程整体结果 |

### 8.2 通知内容

- 流程名称
- 步骤名称
- 执行时长
- 成功/失败状态
- 错误信息（失败时）

## 九、定时任务配置

项目提供 systemd 用户级定时任务样例：

```
deploy/systemd/user/
```

配置要点：
- `WorkingDirectory` 设置为项目根目录
- `ExecStart` 指定执行脚本
- 使用 `%h` 引用用户主目录

---

## 十、AI 助手行为准则

当基于此 Prompt 维护调度模块时，AI 助手应：

1. **保持编排边界**：不添加业务逻辑
2. **维护流程顺序**：不随意调整步骤依赖关系
3. **保护敏感信息**：不在代码中暴露通知配置
4. **支持步骤扩展**：新增步骤通过配置添加
5. **保持通知抽象**：通知客户端设计保持可扩展
